# Issue Format

Use this structure for each meaningful issue.

```text
Problem
- Clear description.

Why It Matters
- Production risk, maintainability risk, user impact, or security/data impact.

Options
- Option A: ...
  Effort:
  Risk:
  Impact:
  Maintenance cost:
- Option B: ...
  Effort:
  Risk:
  Impact:
  Maintenance cost:
- Option C: Do nothing.
  Effort:
  Risk:
  Impact:
  Maintenance cost:

Recommendation
- Choose one option and explain why.
```

Keep the issue concise. Do not include generic advice that does not change a decision.
